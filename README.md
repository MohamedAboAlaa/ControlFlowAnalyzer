# Control Flow Syntax Analyzer & Parser

A compiler front-end system built with **ANTLR4** and **Java** that tokenizes, parses, validates, and generates an Abstract Syntax Tree (AST) for structured control flow constructs: `if`, `if-else`, and `for` statements — including deeply nested combinations.

## Project Overview

This project implements the **lexical analysis** and **syntax analysis** stages of a compiler pipeline. Given a snippet of source code containing control flow statements, the system:

1. **Tokenizes** the input (Lexer)
2. **Parses** the token stream against a Context-Free Grammar (CFG) (Parser)
3. **Validates** syntax and reports errors with precise line/column information
4. **Builds an AST** that represents the hierarchical structure of the program

The project is built on top of **ANTLR 4.13.1**, which auto-generates the Lexer, Parser, and Visitor base classes from the grammar file `ControlFlow.g4`.

---

## Features

- ✅ Tokenization of keywords, identifiers, numbers, operators, and delimiters
- ✅ Recursive grammar supporting arbitrarily nested `if`, `if-else`, and `for` structures
- ✅ Automatic **Dangling Else** resolution (via ANTLR's greedy matching)
- ✅ Clean syntax error reporting with line and column numbers
- ✅ AST generation with a custom Visitor pattern
- ✅ Human-readable tree printer with depth-based indentation
- ✅ Support for single-statement and block (`{ }`) bodies
- ✅ Optional `int` type in `for` initializer (`int i = 0` or `i = 0`)
- ✅ Support for `i++`, `i--`, and assignment updates in `for` loops

## Architecture & Pipeline

```
Source Code (String)
       │
       ▼
  ControlFlowLexer          ← Splits input into tokens
       │
       ▼
  CommonTokenStream          ← Token buffer
       │
       ▼
  ControlFlowParser          ← Matches tokens against grammar rules
       │                        (attaches SyntaxErrorListener)
       ▼
    ParseTree                ← ANTLR's Concrete Syntax Tree
       │
       ▼
 ControlFlowASTVisitor       ← Walks parse tree, creates ASTNodes
       │
       ▼
      AST                    ← Our simplified Abstract Syntax Tree
       │
       ▼
   printAST()                ← Pretty-prints tree with dash-indentation
```

## Grammar Design (ControlFlow.g4)

The grammar is a **Context-Free Grammar (CFG)** written in ANTLR4 syntax.

### Parser Rules

| Rule | Description |
|---|---|
| `program` | Top-level entry; zero or more statements followed by EOF |
| `statement` | Dispatches to `ifStatement`, `forStatement`, `block`, or `simpleStatement` |
| `ifStatement` | Two alternatives: `IfOnly` (no else) and `IfElse` (with else) — labeled for the Visitor |
| `forStatement` | Full `for(init; condition; update) statement` construct |
| `block` | `{ statement* }` — zero or more statements in braces |
| `simpleStatement` | `IDENTIFIER = expression ;` or bare `expression ;` |
| `condition` | An expression optionally followed by a relational operator and another expression |
| `init` | Optional `int` type, then `IDENTIFIER = expression`; may be empty |
| `update` | `IDENTIFIER++/--` or `IDENTIFIER = expression`; may be empty |
| `expression` | Binary arithmetic (`+`, `-`, `*`, `/`), identifier, or number literal |

### Lexer Rules

| Token | Pattern | Examples |
|---|---|---|
| `IF`, `ELSE`, `FOR`, `INT_TYPE` | Keywords | `if`, `else`, `for`, `int` |
| `IDENTIFIER` | `[a-zA-Z_][a-zA-Z_0-9]*` | `x`, `myVar`, `i` |
| `NUMBER` | `[0-9]+` | `0`, `42`, `100` |
| `RELOP` | `<`, `>`, `<=`, `>=`, `==`, `!=` | `x < 10`, `i == 3` |
| `INCOP` | `++` or `--` | `i++`, `j--` |
| `ASSIGN` | `=` | `x = 5` |
| `PLUS/MINUS/STAR/SLASH` | `+`, `-`, `*`, `/` | Arithmetic |
| `LPAREN/RPAREN` | `(`, `)` | |
| `LBRACE/RBRACE` | `{`, `}` | |
| `SEMI` | `;` | Statement terminator |
| `WS` | Whitespace | Skipped silently |
| `LINE_COMMENT` | `// ...` | Skipped silently |

## File-by-File Reference

### `ControlFlow.g4`
The heart of the project. Defines the full CFG for the supported language. The `@header` block ensures all generated Java files belong to `package com.controlflow`. The `ifStatement` rule uses ANTLR **alternative labels** (`# IfOnly`, `# IfElse`) to let the Visitor distinguish between the two forms without manual type-checking.

### `ASTNode.java`
Base class for all AST nodes. Keeps the design intentionally simple:
- `label` (String) — the human-readable name shown when printing
- `children` (List\<ASTNode\>) — child sub-trees
- `addChild(ASTNode)` — null-safe child attachment

All concrete node types are **static inner classes** that call `super(label)`. This avoids the need for generics or abstract methods and makes the tree trivially printable by a single recursive method.

### `ControlFlowASTVisitor.java`
Extends the ANTLR-generated `ControlFlowBaseVisitor<ASTNode>`. Each `visitXxx` method:
1. Creates the appropriate `ASTNode` subtype
2. Recursively visits child parse-tree nodes
3. Attaches resulting child `ASTNode`s
4. Returns the constructed node

Key methods: `visitProgram`, `visitIfOnly`, `visitIfElse`, `visitForStatement`, `visitBlock`, `visitSimpleStatement`.

### `SyntaxErrorListener.java`
Extends ANTLR's `BaseErrorListener`. Overrides `syntaxError()` to collect formatted error strings (`"line L:C — message"`) into a list instead of printing to stderr. `Main.java` checks `hasErrors()` after parsing and prints all collected errors if any exist.

### `Main.java`
Orchestrates the entire pipeline:
1. Defines the input string
2. Creates `ControlFlowLexer` → `CommonTokenStream` → `ControlFlowParser`
3. Attaches `SyntaxErrorListener` (replaces ANTLR's default stderr listener)
4. Calls `parser.program()` to start parsing
5. If errors exist, prints them and exits
6. Otherwise, runs `ControlFlowASTVisitor` to produce the AST
7. Calls `printAST()` for a depth-indented tree display

### `pom.xml`
Maven build file with three plugins:
- **antlr4-maven-plugin** (4.13.1) — reads `ControlFlow.g4`, generates Lexer/Parser/Visitor Java sources into `target/generated-sources/antlr4`; `<visitor>true</visitor>` is required for our Visitor pattern
- **maven-compiler-plugin** — compiles all Java sources (hand-written + generated) targeting Java 11
- **maven-jar-plugin** — packages everything into a runnable JAR with `com.controlflow.Main` as the entry point

## AST Node Types

| Node Class | Label | Description |
|---|---|---|
| `ProgramNode` | `Program` | Root; wraps all top-level statements |
| `IfNode` | `If` | Represents `if(cond) stmt` |
| `IfElseNode` | `IfElse` | Represents `if(cond) stmt else stmt` |
| `ForNode` | `For` | Represents a `for` loop |
| `BlockNode` | `Block` | Represents a `{ ... }` block |
| `ConditionNode` | `Condition: <text>` | Holds the raw condition expression text |
| `InitNode` | `Init: <text>` | Holds the for-loop initializer text |
| `UpdateNode` | `Update: <text>` | Holds the for-loop update expression text |
| `ThenNode` | `Then` | Wraps the then-branch of an if |
| `ElseNode` | `Else` | Wraps the else-branch of an if-else |
| `BodyNode` | `Body` | Wraps the body of a for loop |
| `SimpleStmtNode` | `Stmt: <text>` | Represents a simple assignment/expression |

## Key Challenges & How They Are Solved

### 1. Dangling Else Problem
**Problem:** In `if (a) if (b) X else Y`, which `if` does the `else` belong to?

**Solution:** ANTLR resolves this automatically through **greedy (eager) matching** — it always associates the `else` with the nearest preceding `if`. The grammar defines `IfElse` as the second alternative of `ifStatement`, and ANTLR's priority rules mean an `else` token is consumed by the innermost `if` first. No additional grammar predicates or disambiguation are needed.

### 2. Nested Structures
**Problem:** Loops and conditions can appear inside each other to any depth.

**Solution:** The `statement` rule is **recursive** — it allows any statement (including `ifStatement` and `forStatement`) as the body of another statement. This mutual recursion is handled naturally by ANTLR's LL(*) parsing strategy.

### 3. Scope Boundaries (Single vs Block Statements)
**Problem:** A body can be either a single statement (`if (x) y = 1;`) or a braced block (`if (x) { y = 1; z = 2; }`).

**Solution:** The `statement` rule includes both `simpleStatement` and `block` as alternatives. The Visitor handles both uniformly — `visitBlock` creates a `BlockNode` with multiple children, while `visitSimpleStatement` creates a single `SimpleStmtNode`.

### 4. Optional For-Loop Clauses
**Problem:** `for` loops may have empty init or update clauses.

**Solution:** Both `init` and `update` grammar rules include an **empty alternative** (`| /* empty */`). The Visitor detects empty text via `.getText().isEmpty()` and substitutes the label `"(empty)"` for display clarity.

## Getting Started

### Prerequisites

- Java 11 or higher
- Maven 3.6+

### Clone & Build

```bash
# Build and generate ANTLR sources
mvn clean compile

# Run the analyzer
mvn exec:java -Dexec.mainClass="com.controlflow.Main"
```

> **Important:** You must run `mvn generate-sources` (or `mvn compile`) **before** opening the project in an IDE, so the ANTLR-generated files (`ControlFlowLexer`, `ControlFlowParser`, `ControlFlowBaseVisitor`) are present and the IDE can resolve them.

---

## Running the Project

Edit the `input` string in `Main.java` to test different programs:

```java
String input = "if (x < 10) { y = x + 1; }";
```

Then rebuild and run:

```bash
mvn compile exec:java -Dexec.mainClass="com.controlflow.Main"
```

## Testing

Test cases are located in `ControlFlowParserTest.java`. Run them with:

```bash
mvn exec:java -Dexec.mainClass="com.controlflow.ControlFlowParserTest"
```

The test suite covers:

| Category | Examples |
|---|---|
| Simple if | `if (x < 10) { y = 1; }` |
| If-else | `if (x == 0) { y = 1; } else { y = 2; }` |
| For loop | `for (int i = 0; i < 5; i++) { x = i; }` |
| Nested if inside for | `for(...) { if(...) { } }` |
| Deeply nested | `if` inside `if` inside `for` |
| Dangling else | `if(a) if(b) x=1; else x=2;` |
| Empty for clauses | `for(;;) { x = 1; }` |
| Multiple top-level stmts | Several statements in sequence |
| Invalid syntax | Missing `)`, `}`, `;` — expects errors |
| Empty block | `if (x < 5) { }` |
| Arithmetic expressions | `y = x + 1 * z;` |
